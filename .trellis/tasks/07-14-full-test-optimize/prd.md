# PRD — 小蚕平台全量功能测试与优化报告

## 背景
小蚕活动平台（前后端分离：后端 Spring Boot，前端 Vue3 + Vite）已部署于 `http://xiaocan.20030704.xyz/`，
含 8 个功能模块。需对其做一次全量功能测试，覆盖前端页面与后端接口，定位可用性/正确性/性能/风控相关问题，
产出优化报告，用于指导后续优化该项目。

## 测试目标
- 验证 8 个模块的页面加载、数据读取、状态变更操作可用性（允许实际触发改变状态的操作）。
- 前后端同时观测：前端通过 Browser Relay 驱动真实登录态 Chrome；后端通过查看接口响应、日志、控制台/网络捕获。
- 页面数据有时需多次刷新才出现，测试需容忍并记录该现象（视为待优化点而非误报）。
- 抢单模块需实际触发，但需评估风控/封号/真实下单风险后再决定触发深度。

## 测试范围（8 模块）
| 模块 | 路由 | 对应后端 Controller | 说明 |
|------|------|---------------------|------|
| 首页/活动列表 | `/` | XiaoChanController | 门店活动列表、搜索、排序、地址过滤 |
| 地址管理 | `/location` | LocationController、LocationPushTargetController | 地址增删改、推送绑定 |
| 监控管理 | `/monitor` | MonitorController | 监控配置（门店关键字、自定义规则） |
| 抢单 | `/grab` | GrabController | 抢单/定时抢单配置、饭票校验 |
| 抢单记录 | `/grab-history` | TaskExecHistoryController | 抢单执行历史 |
| 卡券查询 | `/grab-card` | GrabController | 饭票/卡券数量查询 |
| 通知记录 | `/notify-history` | NotifyHistoryController | 推送通知历史 |
| 设置 | `/settings` | ProxyConfigController、UserController | 代理IP池、登录态、账号管理 |

## 约束
- 不在测试中破坏线上数据完整性（删除操作需谨慎，优先用可恢复的测试数据）。
- 抢单触发需先评估风控；若实际抢单下单，需用户确认。
- 测试以「找出优化点」为目的，不只做 happy-path 验证。

## 验收标准
- [ ] 8 个模块均被测试，每模块产出：页面加载情况、关键操作实测结果、发现的问题。
- [ ] 前后端双侧观测：前端页面行为 + 后端接口响应/状态均记录。
- [ ] 产出一份结构化优化报告（含问题清单按模块分类、严重度、复现步骤、优化建议）。
- [ ] 「需多次刷新才出数据」等不稳定现象单独归类分析。
- [ ] 抢单模块的触发深度与风险在报告中明确说明。
